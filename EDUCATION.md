## 2. From rules to learning   
`RULES → LEARNING`

> **Slice (animation):** technique=canvas — Single full-width <canvas> (CSS height ~320px; set canvas.width/height to clientWidth*devicePixelRatio and ctx.scale(dpr,dpr) for crispness). Read theme colors once per frame via getComputedStyle(document.documentElement).getPropertyValue('--ink') for line/text, '--accent' (class A) and '--accent-2' (class B) for the two point colors, '--faint' for the boundary, '--reject' for the misclassification rings, and '--bg' to clear — these are all defined in the harness style.css for both the paper and luminous (data-theme=dark) modes, with distinct values, so it reads in both with NO hardcoded hex and NO external libraries. Animate one variable t in [0,1] via requestAnimationFrame: dt=(now-last)/1400, t=min(1,t+dt). Phase the timeline: RULES = t in [0,0.4], DISSOLVE = [0.4,0.6], LEARN = [0.6,1.0].
> 
> Setup once: define a hidden 'true' wavy boundary y_b(x)=cy + 46*sin(x/W*3.2) (W=canvas CSS width, cy=center y). Generate 28 fixed data points with a seeded LCG (seed=12345; rng=()=>((seed=(seed*1103515245+12345)&0x7fffffff)/0x7fffffff)). For each point: pick x,y in the canvas, then assign cls = (point.y < y_b(point.x)) ? A : B, with a SMALL per-point noise margin so a few points sit near the boundary — but assign the label from y_b itself so every point is, by construction, on the correct side of the learned boundary y_b (the learned rule is consistent with the labels). Store {x,y,cls}. Also precompute 7 'rule segments': axis-aligned horizontal+vertical staircase strokes (random start/len) that crudely approximate the boundary — the hand-written rulebook. After generating, find 3 points that the staircase puts on the WRONG side (i.e., the staircase's piecewise region disagrees with cls) while y_b classifies them correctly; mark these as the misclassified-by-rules set. This guarantees the LEARN phase can honestly flip them to correct.
> 
> Draw per frame (clear with --bg):
> 1) RULES phase: draw the 7 staircase segments in --ink, lineWidth 2, with a slight hand-jitter (offset each endpoint by sin(seed_i+now/300)*1.5). Fade them in via globalAlpha=clamp(t/0.1). Render small 'IF/THEN' mono labels (ctx.font in a monospace, fillStyle --faint) beside two of them. Draw a tiny --reject ring around each of the 3 staircase-misclassified points to show the rulebook's errors.
> 2) DISSOLVE phase: ramp segment alpha down = clamp((0.6 - t)/0.2), and let each staircase vertex drift toward the nearest data point (lerp by ease=(t-0.4)/0.2) so the rules visually 'melt' into data.
> 3) Always draw the 28 points (alpha clamp((t-0.35)/0.1)): filled circles r=5, class A in --accent, class B in --accent-2, 1px --bg stroke for separation.
> 4) LEARN phase: draw the learned boundary as a polyline sampled across x (step 6px) of y_b(x), but reveal it progressively — only draw up to x_max = W*clamp((t-0.6)/0.35,0,1), lineWidth 2.5, strokeStyle --faint, with a soft 'settling' wobble amp=6*(1-progress) added to y so it eases into place. Because every point's label was assigned from y_b, the finished boundary genuinely separates the 28 points. After t hits 1, recolor the 3 previously-ringed points to their correct class color and drop the --reject ring, showing the fit corrected the rulebook's errors.
> 
> Easing: use ease=x*x*(3-2*x) (smoothstep) on phase-local progress. Stop the rAF loop when t>=1 and a 600ms hold has elapsed; expose a replay() that resets t=0. Keep total cost trivial (~40 primitives/frame). _(interaction: On hover/click the canvas, toggle between two states: the static hand-written staircase ('rules', showing the 3 misclassified points ringed in --reject) and the fitted smooth boundary ('learned', rings dropped and those points on the correct side) — letting the visitor flip the same data between a coded rule and a learned one. A click also restarts the dissolve-to-learn animation via replay(). Respect prefers-reduced-motion: skip the rAF tween and render the final LEARN frame directly (boundary drawn, points correctly classified).)_

## 3. Learning a boundary   
`SUPERVISED LEARNING`

> **Slice (animation):** technique=canvas — NO external libraries: only canvas 2D, requestAnimationFrame, a range input, a checkbox, and getComputedStyle. SETUP (run once): seed a deterministic PRNG (mulberry32, fixed seed) so the layout is stable. Generate N=120 points in a canvas W x H. TRUE RULE (low-frequency): class A if y is below a baseline sinusoid, class B if above, using s(x)=0.5*H - 0.35*H*sin(2*pi*x/W). Add label NOISE: flip the class of ~12% of points — these flipped points are the noise a wiggly curve will chase. Split into TRAIN (first 80) and TEST (last 40); store {x,y,cls,isTrain}. COLORS: classA='#2563eb' (blue), classB='#e0533d' (warm red) — both clear ~3.7:1 contrast on white and on a dark background, and classes are ALSO distinguished by shape (circle vs triangle), not color alone. Outline each point with the resolved page text color so points stay visible in any theme. Read getComputedStyle for the text/bg CSS custom properties at DRAW time so a theme switch re-themes the boundary and labels; keep the two class colors fixed. BOUNDARY MODEL (the morph) — use FREQUENCY, not point-wrapping, because the boundary is a single-valued function y=yb(x) and CANNOT loop around an isolated point: SMOOTH baseline = s(x) (equals the true rule, the best generalizer). To raise 'flexibility' t (slider, 0..1), add higher-frequency terms tuned to reduce TRAIN error: yb(x)=s(x)+ amp(t)*sum_{k=1..K(t)} c_k*sin(k*2*pi*x/W + phi_k), where K(t) grows from ~1 to ~20 and amp(t) grows with t, and the c_k,phi_k are fit (e.g. a few steps of coordinate descent at setup, cached per t-bucket) to flip the sign of the boundary's residual at as many TRAIN points as the frequency budget allows. Equivalent simpler implementation: a per-x-bin threshold where bin count grows from 1 to ~80 with t and each bin's threshold is chosen to best classify the TRAIN points in that bin — at high t the boundary resolves individual training points (including noise), at low t it is the smooth rule. (This is the same low-frequency-generalizes / high-frequency-overfits mechanism the harness's own classify gate uses: Ry(x) vs Ry(7x).) DRAW LOOP: clear; for x=0..W step 2 compute yb(x), stroke a 2px polyline in the resolved text color; optionally fill the two regions above/below at ~10% alpha in the class colors. Draw points: predict A if y<yb(x) else B. TRAIN points solid; TEST points drawn only when the 'show test' toggle is on, as hollow squares so they read as unseen. SCORING: trainAcc = fraction of TRAIN points whose predicted class equals its labeled class; testAcc = same over TEST. Show both as text plus two small horizontal bars. KEY BEHAVIOR (verified by simulation of this model): as t goes 0->1, trainAcc climbs monotonically toward ~100% while testAcc rises to an INTERIOR peak at moderate flexibility then FALLS — the high-frequency wiggles help training but hurt unseen points. Compute testAcc across a sweep of t at setup, find the argmax, and draw a faint vertical guide at that t labeled 'best generalization'. IDLE: before first interaction, gently auto-sweep t=0.5+0.5*sin(time*0.4) on requestAnimationFrame; on first slider input, stop the sweep and hand control to the user. Redraw on slider 'input' and on toggle change. _(interaction: A horizontal range slider (0..1, 'flexibility') morphs the boundary from a smooth low-frequency curve to a high-frequency wiggly one that chases individual training points. Live training-accuracy and test-accuracy readouts update as the visitor drags; the test curve visibly turns over once the boundary starts chasing noise. A small toggle reveals or hides the held-out test points.)_

## 5. Networks that learn features   
`FORWARD PASS`

> **Slice (animation):** technique=canvas — Use one <canvas> sized to its container with devicePixelRatio scaling (set canvas.width=cssW*dpr, ctx.scale(dpr,dpr)). Hardcode layer sizes as an array, e.g. layers=[3,5,5,2]. Compute each neuron's (x,y): x = padX + layerIndex*(W-2*padX)/(layers.length-1); y spaces neurons evenly down the column and centers shorter columns. Store fixed random weights w[l][i][j] in [-1,1] seeded by a small LCG so it's deterministic. Read theme colors once via getComputedStyle(document.documentElement).getPropertyValue('--accent') (and '--accent-2','--ink-2','--bg'); if any is empty fall back per variable (accent '#3957c4', accent-2 '#3fe0e6', ink-2 '#8a93a6', bg '#0f1115') and re-read on theme change (MutationObserver watching documentElement's data-theme attribute). Write a small parseColor() that accepts '#rgb', '#rrggbb', and 'rgb()/rgba()' strings (getComputedStyle may return any of these) and returns {r,g,b}; trim whitespace first. Animation state: a single scalar 'wave' in [0, layers.length-1] advanced each rAF frame by dt*speed (speed ~1.2 layers/sec). A neuron in layer L is 'active' = clamp(wave - L + 1, 0, 1) giving a smooth 0->1 ramp; its activation value a[L][i] is computed by a real forward pass at load time: a[0] = fixed input vector, a[L][j] = tanh(b[L][j] + sum_i a[L-1][i]*w[L-1][i][j]) (b = small fixed biases, so the bias mentioned in the copy is real). Draw order per frame: (1) clear with --bg; (2) edges as lines from each neuron to each next-layer neuron — lineWidth = 0.4 + 2.4*|w|*sourceActivation*targetReadiness, strokeStyle = accent with globalAlpha = 0.12 + 0.6*|w|*sourceActivation, where positive weights use --accent and negative weights use --accent-2 so sign is visible; (3) neurons as filled circles radius ~14, fill = linear-RGB mix from --bg toward --accent by (active * |a[L][i]|) so a strongly-firing active neuron glows against the page in either theme, stroke = --ink-2; add a soft glow via shadowBlur = 18*active*|a| with shadowColor=accent, and reset shadowBlur=0 after drawing neurons so edges/text aren't blurred. Because both the neuron fill and the glow are mixed/measured against the live --bg, the diagram keeps contrast in light and dark themes. When wave reaches the last layer, hold ~0.8s, then either loop the forward pass or, if training phase enabled, show a 'target' dot beside each output, draw the numeric loss = sum((a_out-target)^2) as small mono text, run a 0.6s backward highlight (iterate layers right->left tinting edges toward an --ink-2 pulse to suggest the gradient flowing back), then take one crude gradient-descent step (w -= lr * approx_grad, lr~0.08) and recompute activations so the replayed pass visibly reduces the loss number. Keep all text in the page's mono font, color --ink-2, ~11px. No external assets; entire thing is shapes, lines, getComputedStyle, MutationObserver, requestAnimationFrame, and arithmetic — zero libraries. _(interaction: A click (or tap) on the canvas restarts the forward-pass wave from the input layer; clicking a 'train' affordance (or clicking repeatedly) advances one training step so the visible loss number ticks down and the activation pattern shifts. Hovering a neuron shows its activation value as small mono text; with no pointer it simply loops the forward pass automatically.)_

## 6. Attention   
`SELF-ATTENTION`

> **Slice (animation):** technique=canvas — Use one <canvas> sized to devicePixelRatio (ctx.scale(dpr,dpr)). Read theme colors once per frame from getComputedStyle(document.documentElement): fg=getPropertyValue('--fg'), bg=getPropertyValue('--bg'), accent=getPropertyValue('--accent'); fall back to '#222'/'#fff'/'#28489e'. Clear each frame with bg so it reads in both paper and luminous modes. DATA: hardcode 7 tokens e.g. ['The','cat','that','ran','was','very','fast']. Build a fixed 7x7 weight matrix W: for each row i (a token's query), assign w[i][j] = exp(s[i][j]) then normalize the row to sum 1 (softmax), where s[i][j] is a small hand-authored score table chosen to tell a story (e.g. 'was' attends strongly to 'cat'; 'fast' attends to 'was' and 'ran'; every token gives itself moderate weight on the diagonal). No randomness needed — these are static illustrative weights, NOT a real trained model, and the caption must say so. Precompute rowMax[i]=max_j w[i][j]; for display, map each cell to a normalized intensity a[i][j]=w[i][j]/rowMax[i] so the strongest attended cell in a row is fully opaque (a row of 7 softmaxed weights peaks well below 1, so painting raw weight as alpha would render the whole grid nearly invisible — normalize by rowMax instead). LAYOUT: top strip y=24 holds 7 token chips evenly spaced across width, each a rounded rect with the token string centered (measureText for width). Below, a grid occupying a square region: N=7 columns and rows, cellSize=gridW/7, origin at (gx,gy). Cell (i,j) filled with accent color at alpha=a[i][j], so a faint-to-solid heat shows. Draw thin 1px grid lines in fg at alpha 0.15. Label rows on the left and columns on top with the token strings (small mono font). INTERACTION/RENDER: track hoveredToken index from mousemove (hit-test the top chips AND the grid rows). When a token i is hovered: (1) outline its chip with accent, 2px; (2) draw the grid ROW i with a 2px accent border and dim all other rows to alpha 0.25; (3) for each j, draw a bond from the bottom-center of chip i to the top-center of chip j, as a quadratic curve, with lineWidth = 1 + 4*a[i][j] and strokeStyle accent at alpha = 0.2 + 0.8*a[i][j] — so strong attention = thick bright bond. Self-bond (i==j) draws as a small loop above the chip. With nothing hovered, draw all bonds faintly (alpha 0.08) so the full structure is visible at rest. ANIMATION: a single eased parameter t in [0,1] driven by requestAnimationFrame eases bond/cell alphas in over ~250ms when hover changes (lerp current shown intensities toward target), giving a smooth highlight rather than a hard switch. Keep a per-cell displayed-alpha array and ease it toward the target each frame; stop the rAF loop when all deltas < 0.002 to save battery, restart on interaction. MODE TOGGLE: a small clickable label bottom-left switching 'parallel' vs 'recurrent'. In 'recurrent' mode, replace the grid with a single horizontal chain: draw arrows chip0->chip1->...->chip6 left to right, and when token i is hovered, animate a pulse (a filled dot) traveling along the chain from token 0 up to i, taking i hops — visually showing that reaching token i costs i sequential steps. In 'parallel' mode the grid + bonds show every pair resolved in one frame (no traveling dot), making the one-hop path length vs step-by-step recurrence contrast literal. All text via ctx.fillText in the page's mono font; no images, no external libs. _(interaction: Hover (or tap) any token to highlight its attention row in the grid and fan out weighted bonds to the tokens it attends to; click the corner label to toggle between 'parallel' (all-pairs grid, one-hop paths) and 'recurrent' (a left-to-right chain where a pulse must travel step-by-step to reach the hovered token), making the scaling contrast visible.)_

## 9. Pretraining, then post-training   
`TWO STAGES`

> **Slice (animation):** technique=canvas — Single <canvas> sized to its container with devicePixelRatio scaling; one requestAnimationFrame loop driven by a t in [0,1) that loops every ~14s. Read theme by sampling getComputedStyle(document.documentElement).getPropertyValue('--accent') and '--accent-2' and '--fg' and '--bg' once per frame (cheap) so it tracks paper vs luminous automatically; draw all strokes/fills from those vars, never hardcoded colors. LAYOUT: divide width into three zones — left third 'corpus', middle 'base model' (a rounded-rect funnel narrowing rightward), right third 'post-training'. Place two vertical guide labels in --fg at low alpha: 'pretraining' under the left+middle, 'post-training' under the right; a thin vertical --rule divider at x=0.62*W marks the stage boundary. CORPUS PARTICLES: maintain an array of ~120 small rounded rects (token-chips), each with a random y, a slight size jitter, and a hue that randomly picks between --accent and --accent-2 at ~35% alpha (the 'messy/broad' look). Each frame advance their x by a speed proportional to (1 - distanceToFunnelMouth) so they accelerate as they converge, and lerp their y toward the funnel centerline (H*0.5) as x crosses into the middle zone — this is the 'funnel'. When a chip reaches the funnel mouth, recycle it to a fresh random y at the far left. BASE MODEL BLOB: at the funnel mouth draw a soft radial-gradient disc (center --accent at ~0.5 alpha fading to transparent) that gently pulses radius with a sin(t*2pi). REFINEMENT: at x=0.62*W draw a 'comb' of ~7 short vertical tick marks in --accent-2; particles that pass the boundary get their y snapped via easing toward one of ~5 evenly-spaced horizontal 'lanes' (ordered rows) and their alpha raised to ~0.85 and color forced to --accent-2 — visually: diffuse cloud on the left of the divider becomes neat parallel rows on the right. Draw a single 'assistant' output node at far right (filled circle, --accent-2) that emits a faint expanding ring each time a particle arrives.
